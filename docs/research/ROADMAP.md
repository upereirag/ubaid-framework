# Roadmap

## Overview

The UBAID Framework develops in versioned increments. Each version produces a specific deliverable and resolves a defined scope of open questions. Earlier versions establish foundation; later versions extend and formalize.

The roadmap is itself subject to revision based on community feedback. Major restructuring proposals should be raised in GitHub Discussions.

---

## Published

### v0.1 — Foundational Taxonomy

**Status:** Published

Established the framework's philosophical foundation: the irreversibility thesis, the Catman concept, the three pillars plus one dimension structure, the Asimov inversion, historical analogies (Gutenberg, Industrial Revolution, Manhattan Project), the gravitational physics model, and the three-level governance architecture.

### v0.2 — Observable Indicators

**Status:** Current — Draft for community review

Establishes the observable indicator system: agent indicators by race (Human, Catman, Autonomous AI), vector indicators for all 14 catalogued illegal celestial bodies with their satellite catalogue, the Black Hole category, the cross-compilation matrix linking agent × vector combinations, and the operational flow (four layers, responsibility chain). Formalizes the distinction between the Astronaut as occurrence and the Catman Agent as legal-universe inhabitant. Renames threshold concepts as Universe Threshold and Entity Threshold for disambiguation.

## In Development

### Block D.1 — Vector Recognition for Layer 1 Automation

**Status:** In parallel development

Structured indicators specifically formatted for automated detection and classification by Layer 1 (AI pre-triage). Integrates with MITRE ATT&CK, NIST, and other existing corpora. Allows the framework to be operationalized in production security environments.

### v0.3 — Entity Threshold Detection Methodology

**Status:** Next milestone

Formal methodology for detecting when an Autonomous AI has crossed the Entity Threshold. Addresses the open question of whether the threshold is operationally detectable or remains a circular definition. Develops indicators of transition, protocol changes when the threshold is crossed, and the formal mathematical conditions that distinguish containable from uncontainable.

## Planned

### v0.4 — Legal Framework Alignment

Maps the UBAID Framework against existing legal and regulatory frameworks: GDPR (Europe), ISO 27001 (international), the EU AI Act, NIST Cybersecurity Framework, and equivalent instruments in other jurisdictions. Establishes how the framework's culpability calculations translate into existing legal categories of liability.

### Software Specification

Reference implementation architecture. Defines the technical requirements for software that implements UBAID: the calculation engine, the calibration interface, the post-quantum cryptography requirements, the API for ecosystem extension, and the governance structure for Level 2 (Implementation Software) deployments.

### v1.0 — Peer Review Submission

The framework consolidated for formal peer review submission to relevant academic venues (philosophy of technology, AI ethics, cybersecurity research, legal theory). v1.0 incorporates the cumulative feedback from community review during v0.2 through v0.4.

---

## Versioning Principles

- **Patch versions** (v0.2.1, v0.2.2) — corrections, clarifications, no structural changes
- **Minor versions** (v0.3, v0.4) — new framework elements, expanded scope, backward-compatible structure
- **Major versions** (v1.0, v2.0) — formal milestones, peer review submissions, fundamental restructuring after substantial community feedback

Each version is published as a tagged release with the corresponding whitepaper PDF.
